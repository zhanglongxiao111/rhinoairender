# 产品交互体验升级提案：从“渲染器”到“设计助理”

**日期**：2025-12-17  
**作者**：Antigravity (AI Product Manager)  
**目标受众**：开发团队、产品经理、设计决策者

---

## 1. 愿景与定位 (Vision)

**现状诊断**：
目前的 Rhino AI Render 插件在技术底层上已经非常扎实（支持多 Provider、截图流畅、UI 风格专业）。但在交互逻辑上，它目前更像是一个**“带有图形界面的 API 调试器”**。用户需要自己管理 Prompt 措辞、自己计算分辨率、自己处理迭代逻辑。

**改进目标**：
我们需要将产品的定位从**“工具 (Tool)”**升级为**“助理 (Assistant)”**。
核心理念是：**“Don't make me think (不要让我思考技术细节)”**。建筑师在使用此工具时，应专注于**设计语汇、氛围和空间**，而非像素、长宽比或 Prompt 语法。

---

## 2. 三大核心痛点与解决方案 (Key Pain Points & Solutions)

### 2.1 痛点一：Prompt 的“白纸恐惧” (The Blank Canvas Paralysis)

**问题描述**：
面对空白的 `<textarea>`，用户往往感到无从下手。建筑师脑海中有画面（如“安藤忠雄风格的清水混凝土”），但不知道如何将其转化为 AI 能理解的精准 Prompt。

**解决方案：结构化灵感系统 (Structured Inspiration System)**

不再强迫用户手写 Prompt，而是提供**“搭积木”**式的构建体验：

*   **风格预设卡片 (Style Presets)**：
    *   提供 6-8 个高频建筑风格卡片（带缩略图）。
    *   *例如*：`极简主义 (Minimalist)`、`参数化 (Parametric)`、`新中式 (Neo-Chinese)`、`粗野主义 (Brutalist)`。
    *   *交互*：点击卡片，自动填充一组经过调优的 Prompt 关键词（如 "concrete texture, raw material, dramatic shadow"）。
*   **环境/氛围选择器 (Atmosphere Selector)**：
    *   使用直观的图标组。
    *   *例如*：☀️ `清晨 (Morning)`、🌇 `黄昏 (Golden Hour)`、🌧️ `雨夜 (Rainy Night)`、🌫️ `雾气 (Foggy)`。
    *   *交互*：点击图标，自动追加光照和天气相关的 Prompt。
*   **智能修饰词 (Magic Modifiers)**：
    *   提供一组“画质增强”标签供点选：`Unreal Engine 5`, `8k resolution`, `ArchDaily featured`, `Cinematic lighting`。

### 2.2 痛点二：缺乏“迭代”的工作流 (Linear vs. Iterative Workflow)

**问题描述**：
目前的工作流是线性的（设置 -> 生成 -> 结束）。一旦生成了新图，旧图就被推入历史堆栈。但设计往往是**非线性**的，建筑师经常需要基于某张“还不错但有点瑕疵”的图继续深化。

**解决方案：迭代式设计画布 (Iterative Design Canvas)**

*   **“基于此图深化” (Refine from Here)**：
    *   在每一张生成结果上增加操作入口。
    *   *功能*：将当前图片的 Seed 或图像本身（Img2Img）作为下一轮生成的基准。
    *   *场景*：“构图对了，但材质不对” —— 锁定构图（ControlNet/Img2Img），只修改材质 Prompt 重新生成。
*   **情绪板模式 (Pin & Compare)**：
    *   允许用户将满意的生成结果“钉 (Pin)”在界面一侧，不随新生成而消失。
    *   *价值*：方便建筑师在一组方案中进行横向对比，或者作为“情绪板”时刻提醒设计方向。

### 2.3 痛点三：参数的“工程师视角” (Engineering-Centric Parameters)

**问题描述**：
侧边栏暴露了 `1024px`, `Long Edge`, `Aspect Ratio` 等底层参数。这些数字对建筑师来说是“无意义的摩擦”，他们更关心图纸的**用途**。

**解决方案：基于场景的配置封装 (Scenario-Based Configuration)**

隐藏枯燥的数字，暴露“使用场景”：

*   **场景预设 (Scenario Output)**：
    *   📐 **概念草图 (Concept Sketch)**：
        *   *配置*：低分辨率、快速模式、高对比度、手绘风格 Prompt 权重增加。
        *   *用途*：方案初期快速推敲体块。
    *   🖥️ **屏幕汇报 (Screen Presentation)**：
        *   *配置*：1920x1080 (16:9)、标准质量、写实风格。
        *   *用途*：PPT 汇报、客户沟通。
    *   📱 **社交媒体 (Social Media)**：
        *   *配置*：1080x1920 (9:16)、高质量、强调构图感。
        *   *用途*：Instagram/小红书分享。
    *   🖼️ **展板大图 (Exhibition Board)**：
        *   *配置*：4K+ 分辨率、最高质量、精细纹理。
        *   *用途*：最终打印或竞赛展板。

## 3. 分阶段实施路线图 (Implementation Roadmap)

为了不影响当前版本的稳定性，建议分三个阶段迭代更新：

| 阶段 | 重点任务 | 预期效果 |
| :--- | :--- | :--- |
| **Phase 1: 辅助输入** | **实现“风格标签”与“氛围选择器”**。<br>在 Prompt 输入框下方增加 Tag Group。 | 解决“写不出 Prompt”的问题，<br>用户只需点选即可生成高质量图片。 |
| **Phase 2: 场景化** | **重构侧边栏参数区**。<br>引入“场景下拉菜单”，隐藏具体数值（保留“高级模式”开关）。 | 界面更加简洁现代，<br>降低新手用户的上手门槛。 |
| **Phase 3: 深度迭代** | **增强图片卡片交互**。<br>添加“钉住”、“作为底图”功能。 | 真正支持专业建筑设计工作流，<br>增加用户粘性。 |

---

## 4. 结语

这个工具不应该只是一个“渲染器接口”，而应该成为建筑师的**“灵感放大器”**。通过上述交互层面的改进，我们可以在保持瑞士风格极简美学的同时，极大地赋予产品**深度**和**温度**。
